## Motivation

Round 2's pop_adaptive agent (-0.64 avg, rank 8/12) failed because of three identified problems:
1. **Only 67.6% training data**: Only player 0's chains were used for training, wasting the opponent-side data
2. **Conflicting gradients**: Random opponent rotation gave the value network contradictory learning signals
3. **Disrupted stat accumulation**: Switching opponents between sessions prevented the adaptive stats from building up

The curriculum agent was designed to fix all three:
- **Block scheduling**: Train against one opponent for 100 sessions before rotating, allowing stats to accumulate
- **Rehearsal buffer**: Store past training data and mix it into updates to prevent catastrophic forgetting
- **Both-player training**: Record chains for both P0 (training agent) and P1 (opponent) to double the training data

The hypothesis: structured population training with these fixes would outperform pure self-play.

## Architecture

CurriculumAgent has the **identical architecture** to AdaptiveValueAgent -- all changes are in the trainer:

```
ValueNetwork(19 -> 64 -> 64 -> 1)
  - Input: 15-dim game state + 4-dim opponent stats = 19 dimensions
  - Architecture: 2-layer MLP with ReLU, 64 hidden units
  - Parameters: ~6,300
```

The agent class inherits directly from AdaptiveValueAgent with no modifications:


The training proceeds in blocks of 100 sessions:
- Sessions 1-100: vs HeuristicAgent
- Sessions 101-200: vs ValueBasedAgent

## Diagnosis & Findings

The diagnosis uncovered one of the most instructive bugs in the entire project.

### 1. The Catastrophic Bug: Both-Player Chain Collection

The "fix" for pop_adaptive's low data utilization -- training on both P0 and P1 chains -- turned out to be **catastrophically wrong** in pool training.

**In self-play**, training on both players works because:
- Both P0 and P1 are the same policy
- States are on-policy for both players
- Rewards are consistent (P0 reward = -P1 reward, but both are from the same agent's perspective)

**In pool training**, P1 is a frozen opponent:
- P1's states are **off-policy** (generated by a different policy entirely)
- P1's rewards are **opposite sign** (zero-sum game: what P1 wins, P0 loses)
- The value network receives **conflicting gradients**: "this state is worth +X" from P0 vs "this similar state is worth -X" from P1

### 2. Chain Analysis -- Quantifying the Damage

| Metric | P0 Chains | P1 Chains |
|--------|-----------|-----------|
| Entries collected | 1,538 | 1,118 |
| Average loss | 15.40 | 20.94 |
| Reward sign relationship | -- | **100% opposite to P0** |

P1 loss is **36% higher** than P0 loss, confirming the value network struggles to fit off-policy opponent states. All 532 hands analyzed showed opposite-sign rewards between P0 and P1 (as expected in a zero-sum game).

### 3. Per-Player Loss Trajectory

| Training Phase | P0 Loss | P1 Loss | Ratio |
|----------------|---------|---------|-------|
| Early (batches 0-4) | 16.28 | 23.07 | 1.42 |
| Mid (batches 25-29) | 12.77 | 17.50 | 1.37 |
| Late (batches 55-59) | 10.42 | 13.93 | 1.34 |

The P1 loss ratio is consistently 34-42% above P0, meaning the network spends roughly 40% of its gradient budget fighting itself.

### 4. Ablation Study -- Isolating Each Fix

| Variant | Avg Score | Change vs Original |
|---------|-----------|--------------------|
| **P0-only chains (fix the bug)** | **-0.363** | **+1.206** |

**Removing both-player chains was the single most impactful fix**, improving performance by +1.2 chips/round -- dwarfing all other ablations. The P0-only variant outperforms the original by a factor that exceeds the difference between the best and worst agents in Round 2.

## Lessons Learned

### Why "More Data" Backfired

The intuition "training on both players' data doubles the training signal" is correct in self-play but catastrophically wrong in adversarial pool training. In a zero-sum game:

```
P0 wins +5 chips  =>  P0 chain teaches "this state is worth +5"
P1 loses -5 chips  =>  P1 chain teaches "this state is worth -5"
```

The same value network is asked to learn that similar game states are simultaneously worth +5 and -5, creating **destructive gradient interference**. The gradients from P0 and P1 chains partially cancel each other, and the network converges to a confused compromise that performs poorly against everyone.

### The Broader Principle

"More data is better" is one of the most reliable heuristics in machine learning -- but it has a critical caveat in adversarial settings: **the data must be on-policy**. Off-policy data from an opponent in a zero-sum game carries opposite-sign rewards that destroy learning rather than enhance it.

---

## Key Insight

In adversarial zero-sum settings, training on the opponent's experience introduces destructive gradient interference that overwhelms any data volume benefit. On-policy data is non-negotiable.
