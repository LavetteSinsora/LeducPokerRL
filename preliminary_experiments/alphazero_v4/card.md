# alphazero_v4 — Training Stability + Exploration Fixes

## Diagnosis from v1–v3

v3 best checkpoint (ep 90K): avg=−0.869, robust=−0.984.
Despite k=30 rollouts and a fixed opponent pool, the agent still:

1. **Oscillates throughout training** — never converges. Root cause: Q* targets are
   generated by the *same* Q-net being trained (non-stationary bootstrap). Every
   weight update shifts the target, creating a feedback loop.

2. **Plays pure strategies** — raise rates collapse toward uniform or zero.
   Root cause: PIMC rollouts use a near-deterministic rollout policy (T=1.0 for both
   players). The imagined opponent never folds to a raise, so Q*(raise | J) ≈ bad.
   The agent learns never to bluff, confirming bluffs look bad in subsequent rollouts.
   This is a pure-strategy attractor that standard PIMC cannot escape.

3. **Loses to all opponents including heuristic** — AZ v3 ep90K: −0.841 vs heuristic,
   −1.117 vs value_based. ValueBased (simple TD) beats heuristic by +0.320. The PIMC
   framework is not finding the right signal.

The fixes below target each root cause independently, without changing the core
PIMC architecture (preserving Texas Hold'em extensibility).

---

## Hypothesis

Four targeted changes address the three root causes:

### Fix 1 — Target Q-net (addresses bootstrap instability)

Keep a frozen copy `q_net_target` of the Q-net. PIMC rollouts use `q_net_target` to
sample actions and compute Q*. The live `q_net` is updated each step. Every
`target_sync_freq` episodes, copy live → target.

This decouples "what we're training toward" from "what we're currently training,"
eliminating the non-stationary target problem. Standard DQN result: this alone
dramatically stabilizes training.

### Fix 2 — Replay buffer (addresses non-i.i.d. online updates)

Store `(P_t, h_onehot, b_mine, Q*_target, legal_mask)` tuples from recent episodes.
At each training step, sample a minibatch from the buffer rather than training on only
the current episode. The StateEncoder and BeliefNet are still updated online (belief CE
loss from current episode); only the Q-net benefits from the buffer.

Buffer size 50K decisions ≈ ~15K games. Batch size 256.

### Fix 3 — Asymmetric rollout temperature (addresses bluffing discovery)

Introduce `T_self` and `T_opp` as separate config parameters:

```
T_self = 0.5   (near-optimal own play in rollouts)
T_opp  = 2.0   (opponent makes some mistakes)
```

When actor == player_i in a rollout, sample from softmax(Q / T_self).
When actor == opponent_j, sample from softmax(Q / T_opp).

With T_opp=2.0, the imagined opponent folds strong hands ~10-20% of the time. This
means Q*(raise | J) gets a non-zero positive contribution when the opponent folds,
breaking the pure-strategy attractor. The agent discovers bluffing is sometimes
profitable.

T_opp also models the real training opponents (heuristic, value_based, cfr) which are
not Nash-equilibrium players — they do make exploitable mistakes. T_opp=2.0 is a
reasonable model of their exploitability.

### Fix 4 — Entropy bonus on Q-net loss (prevents strategy collapse)

Add an entropy regularization term to the Q-net loss:

```
L_q_total = L_q_mse  −  λ_ent · H(π_legal)
```

where `π_legal = softmax(Q[legal_actions])` and H is entropy.
This directly penalizes collapsing to a pure strategy. λ_ent = 0.01 is small enough
not to distort the value estimates but sufficient to keep the policy mixed.

| Parameter | v3 | v4 |
|-----------|----|----|
| Q* target source | live q_net | **frozen q_net_target** |
| Replay buffer | none (online) | **50K decisions, batch 256** |
| Rollout temperature (self) | T=1.0 | **T_self=0.5** |
| Rollout temperature (opp) | T=1.0 | **T_opp=2.0** |

Architecture unchanged from v3: d=4, state(8,), belief(8,8), Q(32,32), k=30.

## Risk / Sensitivity

- **T_opp too high** (e.g., 5.0): agent learns to exploit random opponents, not real
  ones. If v4 shows high performance vs heuristic but collapses vs cfr, reduce T_opp.
- **Buffer + target net interaction**: if target_sync_freq is too long (>2000 ep),
  stale targets produce incorrect Q* even with the buffer. Start at 500.
- **λ_ent too high** (e.g., 0.1): entropy bonus dominates MSE loss; agent plays
  randomly. Check: loss should be dominated by L_q_mse, not L_entropy.
- **Replay buffer Q* staleness**: Q* was generated by an earlier q_net_target. With
  50K buffer and target sync every 500 ep, the oldest Q* in the buffer is ~100 target
  versions old. This is acceptable (DQN uses far larger ratios). If training
  diverges, reduce buffer size to 10K.
